### Instruction Definition
- A binary or machine-level instruction is a multiple-field binary string to be decoded by the control unit of the processor.
- It represents the interface between HW and SW.
- In general, an instruction consists of:
	- An opcode field describing the operation that has to be performed 
	- Some address fields indicating where the input operands are located and where the result should be stored 
	- (in some old architectures) an equal number of address qualifier fields telling how to access the operand reported in the corresponding address field
### INSTRUCTION SET ARCHITECTURE
- The instruction set architecture (ISA) includes all the rules describing the functional specifications of a given architecture
- Each ISA is characterized by: 
	- Different classes of instruction operators 
	- Number of operands 
	- Different addressing modes, i.e., different mechanisms to locate and to access the operands for a given instruction
### CLASSES OF INSTRUCTION OPERATORS
- All computers provide a full set of operations for the first 3 categories 
- System functions in the instruction set vary widely among architectures 
- The amount of support for the last four categories may vary largely 
	- Floating-point instructions are sometimes part of an optional instruction set 
	- Decimal and string were primitives in old systems, but they are no longer used in nowadays processors 
	- Graphic and multimedia instructions typically operate on many smaller data items in parallel; e.g., performing eight 8-bit additions on two 64- bit operands

# ADDRESSING MODES
#### CLASSIFYING OPERAND ACCESS MODES
- **Stack**: 
	- operands are on the top of the stack, LIFO
		- push A
		- push B
		- add
		- pop C
- **Accumulator**: 
	- one operand in memory, the other and the result in the accumulator 
		- Load A
		- add B
		- Store C, store the content of the accumulator in C
- **Register-memory**: 
	- one operand in memory, the other and the result in two registers of the register file 
		- Load R1, A
		- Add R3, R1, B
		- Store R3, C
- **Register-register**: 
	- both operands have to be placed from memory into registers. The result is also stored in a register
	- all instruction do only one thing
		- Load R1, A
		- Load R2, B
		- Add R3, R1, R2
		- Store R3, C

### ADDRESSING MODES
- Usually, instructions have 2 or 3 operands. In case of two operands, one of them represents also the location where the result should be stored
- **Immediate addressing**
	- The operand is a constant specified within the instruction
	- example at slide 18
- **Register addressing**
	- Required operands can be found in a specified register
	- example slide 19
- **Register indirect**
	- The operand is in the memory location pointed by a register
	- example slide 20
	- Register indirect addressing modes are important because they allow natural and automatic modification of pointers by modifying the address value.
- **Register indirect, autoincrement** 
	- *Autoincrement* or to *autodecrement* the register content (before or after performing the memory access) by a known quantity
	- example slide 21
- **Register indirect, displacement, indexed** 
	- *Increment* or *decrement* the register value by a constant (displacement mode) or by the content of another register (indexed mode)
	- two example on slide 22
- **Memory direct** addressing 
	- The operand is fetched from an explicit address of memory or can be found from a separate data word following the instruction
	- example slide 23
- **PC-relative** addressing 
	- This is a special type of displacement addressing, in which the considered register is the Program Counter 
	- This is particularly useful to calculate the destination address of branch instructions, which are known at compile time

# INSTRUCTION ENCODING

### ASSEMBLY LANGUAGE
- Symbolic representation of the machine language of a specific processor
- Invented to simplify programming 
	- Converted to machine code by an assembler program

### INSTRUCTION ENCODING
- Instruction encoding affects: 
	- the size of the compiled programs 
	- the implementation of the processor, which must decode this representation to quickly find the operation and its operands.
- 3 main encoding schemes
	- **Variable** (in opcode size or in number of operands): suitable when there are many addressing modes and operations. Flexible, but complex decoding logic 
	- **Fixed**: opcode size and number of operands are fixed. Suitable when there are few addressing modes and operations, but they lead to larger programs 
	- **Hybrid**: multiple formats specified by the opcode, adding one or two fields to specify the addressing mode or the operand address

## Implementing a basic RISC CPU
To simplify the analysis, we will assume that: 
- Instructions are few in number and one size only (32 bit): 
	- ALU instructions (direct register, immediate) 
	- Load and store instructions (indirect register, displacement) 
	- Branches (only based on comparison with 0) and jumps (PC-relative, indirect register)
	- All ALU instructions apply to data in registers and typically change the entire register
- **Instruction execution stages**
	- Instruction fetch stage (IF)
		- Send the program counter (PC) to memory and fetch the current instruction from memory or I-cache. Update the PC to next sequential value (e.g., by adding 4 if an instruction is 4 byte long) 
	- Instruction Decode/ register fetch stage (ID)
		- decode the instruction and perform simultaneous operand reading. Instruction decode and operand reading can be done in parallel because RISC instructions are fixed-field encoded.
	- Execution/effective address stage (EX):
		- The ALU operates on the operands prepared in the previous cycle, performing different operations depending on the instruction type.
		- **Load/store instructions**: The ALU adds the base register and the offset to form the effective address 
		- **Register-Register ALU instructions**: The ALU performs the operation specified in the ALU opcode on the values read from the register file 
		- **Register-Immediate ALU instructions**: The ALU performs the operation specified by the ALU opcode on the value read from the specified register and the sign- extended immediate 
		- **Branch instructions**: The ALU adds the next instruction value to the extended value of the address (multiplied by 4) to calculate the branch target address (BTA) 
	- Memory access stage (MEM): 
		- **Load instructions**: The memory performs a read using the effective address computed in the previous cycle using A and Imm and saved in ALU output 
		- **Store instructions**: The memory performs a write using the effective address computed in the previous cycle using A and Imm and saved in ALU output, while the value comes from the other register, saved in B
		- **Branch instructions**: in branches, the PC is replaced with the BTA (saved in ALU output) according to the outcome of the comparison with zero, saved in Cond 
	- Write-back stage (WB)
		- **Loads and ALU instructions**: Write the result into the register file, whether it comes from the memory system (for a load) or from the ALU (for an ALU instruction) 
	- Note that if a non-pipelined implementation is used, different instructions have variable latency, depending on actions actually performed. 
	- It is important to perform approximately the same amount of work in each stage. In fact, the clock cycle of the processor depends on the latency of the stage with the worst-case delay
