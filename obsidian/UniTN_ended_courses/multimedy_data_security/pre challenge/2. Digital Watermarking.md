*At a very general level digital **watermarking** is a way of hiding a message (called stegomessage or watermark) within a host (cover) signal.*

By exploiting “perceptual headroom”
- Human perception is imperfect
- Make modification to the original data without changing its perceptual quality
- Modifications can be detected via signal processing

### Requirements
- Imperceptibility
- Robustness  
	- The embedded data should survive any signal processing operation the host signal goes through and preserve its fidelity
	- we want our watermark to survive to a number of signal processing (second part of the channel) we will test the robustness of the watermark
	- watermarking and compression fight between each other, to compress the data usually the first thing u delete is the watermark because is a useless part of data for the image 
- Capacity
	- Maximize data embedding payload
- Security 
	- Kerckhoffʼs principle states that one should assume that the method used to encrypt the data is known to an unauthorized party and that the security must lie in the choice of a key
	- Active Attacks 
		- Hacker has access to some data and attempts to extract information about secret data 
	- Passive Attacks
		- Hacker tries to get information about secret data without playing with data (eavesdropping) 
- ***Requirements slide 7 important to keep as a reference***
	- its more difficult hide a watermark in a flat and monochromatic part of the image
	- all this requirements may change if u allow to the structure to have access on the data, if u have a blind or a non blind structure
		- if u work with a non blind structure 
- **Robust watermarking** is required in copyright protection and ownership identification scenarios
- a fragile watermark is needed for auth and tamper proofing application 
- a statistically invisible watermark is mandatory for stenography
- a high payload is required for labeling application and legacy systems 

### Classification
- a lot of things 
- blind techniques
	- the watermark is recovered without restoring to the original non-marked or any information delivered from it
- Non-blind techniques 
	- the original image is needed to read the watermark 
	- robustness is more easily achieved 
	- often the application scenario does not allow the decoder to access the original non-marked image 
- Decoding process
	- detectable
	- readable watermark
- Symmetric vs. asymmetric schemes
	- in order to improve security in a public detection scenario asymmetric schemes have been proposed 
	- the detection key is different than (a subset of) the embedding key 
	- detecting the watermark should not help removing it 
- Spatial domain watermarking 
	- Watermark embedded by directly modifying the pixel values 
- Transform domain watermarking 
	- Watermark embedded in the transform domain e.g., DCT, DFT, wavelet by modifying the coefficients of global or block transform
- spatial domain
- transform domain
- hybrid techniques
