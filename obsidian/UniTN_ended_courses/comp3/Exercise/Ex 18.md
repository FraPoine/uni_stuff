what the fuck does it mean 

SUBSET SUM
- we want to find, in a set of integers, a subset whose sum is equal to an integer k 
- if it exists return true, if not false 

lets start by understanding what does mean the task
1. design an algorithm that solves SUBSET SUM on a **Deterministic** device
2. with a O(log n) additional space constraint 

**1** 
- SUBSET SUM è un problema **NP-completo**. I migliori algoritmi conosciuti (es. programmazione dinamica o enumerazione dei sottoinsiemi) richiedono **almeno O(n)** o **O(s)** spazio — molto più di O(log n).
- Con **O(log n) di spazio aggiuntivo**, **non puoi tenere traccia di combinazioni di elementi, somme parziali, o memoization**, nemmeno ricorsione. È un vincolo **troppo severo** per un problema così complesso.
- Infatti:
	- È noto che problemi NP-completi **non possono** essere risolti in **log-space deterministico** (a meno che P = L, cosa che non si crede vera).
- SUBSET SUM è **PSPACE-hard** se generalizzato, ma già difficile per spazi logaritmici.


**2**
- O(n) oppure O(n^2)
