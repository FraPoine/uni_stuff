Consider the UNIVERSITY HIRING decision problem:
- A university needs to hire the teaching staff for a new degree, for which a set T of topics must be taught. The executive board received n applications from prospective teachers, and every applicant i ∈ {1, . . . , n} has knowledge of a subset Si ⊆ T of the required topics. The budget allows for hiring at most k ≤ n teachers.
- Is there a choice of k applicants so that all teaching topics are covered?
- An instance of the problem consists of parameters n, k, T, S1, . . . , Sn.
1) Prove that UNIVERSITY HIRING ∈ NP.
2) Prove by reduction that UNIVERSITY HIRING is complete in the class NP. The reduction can refer to any language discussed during the course.
3) Prove that if k is kept constant (e.g., k = 10), then the problem’s asymptotic complexity is polynomial wrt input size.

## **To prove: UNIVERSITY HIRING ∈ NP**
This means:
> If the answer is **YES**, there exists a **certificate (witness)** that proves it, and we can **verify** the correctness of that certificate in **polynomial time**.

### Step 1: Define the certificate (witness)
If the answer is "yes", then a valid certificate is:
- A list of at most `k` applicants: say `A = {i₁, i₂, ..., i_k}`.
This list tells us **which applicants to hire**.

### Step 2: Verification algorithm
We now describe a polynomial-time algorithm to verify that the certificate is valid.
Input: `(n, k, T, S₁, ..., Sₙ)` and the certificate `A = {i₁, ..., i_k}`.
Algorithm:
1. **Check that the size of A is at most k.**
2. **Compute the union of the subsets** `S_{i₁} ∪ S_{i₂} ∪ ... ∪ S_{i_k}`.
3. **Check if this union is equal to T**.
If all three conditions are satisfied, **accept** the certificate (i.e., return "yes"), else **reject**.

### Step 3: Time complexity of verification
Let’s analyze the time needed:
- Checking the size of A: `O(1)`
- Union of ≤ k subsets, each subset ⊆ T (say, |T| = m):  
    → Union takes **O(k * m)** time.
- Comparing the union to T:  
    → Also **O(m)** time (assuming sets are represented with bitsets or hash sets).
So, the total time is **polynomial in the input size** (depends on `n`, `k`, and `|T|`).

:) UNIVERSITY HIRING ∈ NP

UNIVERSITY HIRING is just a rephrasing of SET COVER. Formally, given an instance of SET COVER, we map the union of all sets to T , and each of the sets to a different Si

### SET COVER (decision version)
**Input**:
- A finite set (called the **universe**) `U = {e₁, ..., e_m}`.
- A collection of subsets `S = {S₁, ..., Sₙ}` where each `Sᵢ ⊆ U`.
- An integer `k`.

**Question**:  
Does there exist a subcollection of at most `k` subsets whose union is equal to `U`?

### Claim: The SET COVER instance is a **yes-instance** if and only if the UNIVERSITY HIRING instance is a **yes-instance**.

#### (⇒) Suppose the SET COVER instance is "yes":
Then, there exist at most `k` subsets whose union covers `U`.
→ That means there are ≤ `k` applicants whose known topics cover all of `T = U`.
→ The UNIVERSITY HIRING instance is also "yes".
#### (⇐) Suppose the UNIVERSITY HIRING instance is "yes":
Then, there are ≤ `k` applicants such that their combined knowledge covers `T = U`.
→ The corresponding subsets from SET COVER form a cover of `U` with at most `k` subsets.
→ The SET COVER instance is also "yes".
✅ So the reduction **preserves the answer**.

## Step 3: Polynomial-time construction
- The topics and subsets are directly copied — no exponential work.
- The reduction runs in **polynomial time**.
#### UNIVERSITY HIRING is **NP-complete**
